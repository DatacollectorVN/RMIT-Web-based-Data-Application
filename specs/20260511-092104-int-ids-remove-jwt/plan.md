# Implementation Plan: Integer primary keys & remove JWT requirement

**Branch**: `20260511-092104-int-ids-remove-jwt` | **Date**: 2026-05-11 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/Users/nhan.ngo/rmit/RMIT-Web-based-Data-Application/specs/20260511-092104-int-ids-remove-jwt/spec.md`

## Summary

Migrate the PostgreSQL schema and application from **UUID primary keys** to **64-bit integer** database-generated primary keys across `users`, `products`, `photos`, `reviews`, `orders`, and `order_items`. Update seed data, SQLAlchemy models, scripts (`migrate.py` unaffected structurally; SQL files change), and OpenSearch mapping so `product_id` is numeric. Remove **JWT as a requirement** for the Phase 1 API: drop mandatory JWT env vars, remove `python-jose` if unused, delete or stub auth middleware dependencies, and amend **SPEC-00 (constitution)** to v2.1 so locked decisions match this feature.

## Phased implementation breakdown

### Phase A — Constitution & docs (blocking)

1. Edit `.specify/memory/constitution.md`: bump **2.0 → 2.1**; replace UUID/JWT locked rows; update §7–§9 examples; sync impact report at top.
2. Update `.env.example`: remove or comment `JWT_SECRET` / `JWT_EXPIRY_HOURS` as optional for Phase 2.
3. Update `README.md` to state integer IDs and public API for Phase 1.

### Phase B — SQL migrations (`app/migrations/`)

1. Replace migration chain **or** add a single “breaking” migration set that drops dependent tables and recreates with `BIGINT GENERATED ALWAYS AS IDENTITY` / `BIGSERIAL` PKs and `BIGINT` FKs (order: extensions → users → products → photos → reviews → orders → order_items → triggers).
2. Ensure `updated_at` triggers still fire.
3. Verify `make migratedbreapply` applies cleanly on empty DB.

### Phase C — Seed data

1. Rewrite `app/scripts/snapshot_seed.sql` with **integer** literals for all PKs/FKs; preserve realistic relationships (orders reference users/products).
2. Run `make snapshotdb` after migrations.

### Phase D — SQLAlchemy models

1. Change all `UUID(as_uuid=True)` PKs/FKs to `BigInteger` with `Identity()` or `autoincrement=True` as appropriate for PostgreSQL.
2. Remove default `uuid.uuid4()` from models; rely on DB identity.
3. Fix relationships and any `str(id)` assumptions.

### Phase E — OpenSearch

1. Add `products_mapping.v2.json` with `product_id`: `{ "type": "long" }`.
2. Point `products_mapping.json` to v2 or duplicate content; update `init.sh` if index must be recreated.
3. Update `reindex_products.py` to emit integer `product_id` and use string int for bulk `_id` if needed.
4. Update `search_relevance_check.py` if queries assume keyword type (usually unchanged for match queries).

### Phase F — JWT removal

1. Remove `python-jose` from `pyproject.toml`; `uv lock` / commit `uv.lock`.
2. Delete unused `middleware/auth.py` stubs or `get_current_user` if present; ensure no router imports them.
3. Grep repo for `JWT`, `jose`, `Bearer`; clean dead code and docs.

### Phase G — Validation

1. `docker compose up -d` → `make migratedbreapply` → `make snapshotdb` → `make opensearchinit` → `make reindexproducts`.
2. `curl` health + five calls without `Authorization` header.
3. Spot-check OpenSearch document: `product_id` is numeric.
