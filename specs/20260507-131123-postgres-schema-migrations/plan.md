## Phase 0: Outline & Research

### Decisions derived from `sample_ddls.sql`

- **D0.1 UUID extension**: `uuid-ossp` is required if any DB-side UUID default remains. If we remove UUID defaults entirely, the extension may become optional; keep it for compatibility unless it causes issues.
- **D0.2 Deletion rules**: `reviews` and `orders` use `ON DELETE CASCADE` from user/product; `order_items` uses `ON DELETE RESTRICT` for product. Keep as-is from sample DDL unless later specs require alternative.
- **D0.3 updated_at behavior**: Use a single `trigger_set_updated_at()` function and per-table triggers for tables that have `updated_at`.

**Open item resolution**: No NEEDS CLARIFICATION required—sample DDL provides concrete constraints, and constitution provides naming and migration structure.

## Phase 2: Action items (step-by-step build plan)

1. **Create migration files** under `/Users/nhan.ngo/rmit/RMIT-Web-based-Data-Application/migrations/` matching the numbering above.
2. **Translate DDL**:
   - Move each `CREATE TABLE` + its indexes into its table migration.
   - Move trigger function + triggers into the final trigger migration.
3. **Align ID strategy with SPEC-00**:
   - Define `id UUID PRIMARY KEY` without relying on database-generated values for correctness.
4. **Apply/rollback drill**:
   - From a clean DB, apply all migrations.
   - Roll back the last migration, then reapply.
   - Attempt representative invalid inserts to confirm constraints (rating bounds, foreign keys, restrict/cascade).
5. **Document** the runbook and update quickstart.

### Makefile workflow (developer ergonomics)

Add a repository root `Makefile` to standardise common schema tasks so developers can run:

- `make migratedb` — apply all migrations using `DATABASE_URL`
- `make migratedb-down-1` — roll back one migration

This reduces copy/paste errors and makes onboarding faster (supports SC-001/SC-002).
